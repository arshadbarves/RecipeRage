## Executive Summary

RecipeRage is a mobile-first, online multiplayer cooking competition game that brings the chaotic joy of party cooking games to players anywhere in the world. Built with esports-grade networking infrastructure and featuring deep progression systems, RecipeRage transforms couch co-op into competitive online battles that are fair, fun, and endlessly replayable.

The cooking game market represents a $1.2-2.4 billion opportunity growing at 8.9% CAGR, yet the dominant franchise (Overcooked) suffers from critical online multiplayer deficiencies that leave millions of potential players frustrated. RecipeRage addresses this gap with a technically superior solution built on Unity's Netcode for GameObjects and Epic Online Services, delivering low-latency, region-optimized matchmaking that existing competitors cannot match.

## Core Vision

### Problem Statement

Overcooked revolutionized the party cooking game genre but remains locked to local multiplayer or plagued by broken online infrastructure. Overcooked 1 offers no online play whatsoever, while Overcooked 2 suffers from:
- Unplayable lag in 80-90% of online matches
- No region selection or ping filtering
- Globally mismatched host connections
- Cross-play connection errors
- No mobile availability

Players seeking chaotic cooking competition with friends online currently have zero viable options. Third-party workarounds (Parsec, Steam Remote Play) introduce latency and friction that destroy the fast-paced gameplay experience.

### Problem Impact

**For Players:**
- Distance-separated friends cannot experience cooking game chaos together
- Mobile users have no access to competitive cooking games
- Players waste time in broken matchmaking with unplayable lag
- The genre's full potential remains untapped due to technical limitations

**For the Market:**
- A $1.2-2.4B cooking game market growing at 8.9% annually
- Mobile gaming driving dominant growth (8.1B global mobile subscriptions)
- Asia-Pacific showing 12-15% CAGR - highest growth region
- Party games trending on mobile but no quality cooking competition exists

**Current Workarounds Fail:**
- Screen-sharing solutions introduce unacceptable latency
- Overcooked 2's netcode is fundamentally broken
- Mobile cooking games offer only single-player or poor multiplayer
- Players give up rather than endure frustrating experiences

### Why Existing Solutions Fall Short

| Competitor | Their Approach | Critical Failure |
|------------|----------------|------------------|
| **Overcooked (PC/Console)** | Local-only couch co-op | No online play, no mobile |
| **Overcooked 2** | Added online multiplayer | 80-90% match lag, no region filter, broken cross-play |
| **Mobile Cooking Games** | Single-player focus | No competitive multiplayer, poor social features |
| **Parsec/Remote Play** | Screen sharing workaround | High latency, requires PC host, clunky mobile controls |

**Root Cause:** Existing solutions were not built with online multiplayer as a first-class priority. Overcooked 2's netcode is an afterthought. No competitor has invested in esports-grade networking infrastructure for the cooking genre.

### Proposed Solution

RecipeRage delivers the first truly competitive-grade, mobile-first online cooking game through:

**Technical Foundation:**
- Unity 6.0 with Netcode for GameObjects (NGO) for authoritative server-client architecture
- Epic Online Services (EOS) for P2P networking, matchmaking, and cross-platform capabilities
- VContainer dependency injection + UniTask async programming for clean, maintainable code
- Region-based matchmaking with ping filtering (solves Overcooked's #1 complaint)
- Smart bot filling system - matches start immediately with AI, no waiting

**Gameplay Innovation:**
- 2-8 player competitive cooking battles (2-4 humans + AI filling)
- Character ability system (Chef Swift, Speedy, Balanced, Strong) for strategic depth
- Complete station-based cooking (Ingredients → Cutting → Cooking → Serving)
- Dynamic order system with time pressure and combo scoring
- Fair progression without pay-to-win mechanics

**Platform Strategy:**
- Mobile-first design (iOS/Android) with touch-optimized controls
- Cross-platform architecture ready for PC/console expansion
- Always-online multiplayer - no scheduling required
- Friends system with P2P chat integration

### Key Differentiators

1. **🎯 Mobile-First + Competitive Grade**
   - First cooking game built for mobile with AAA networking standards
   - Technical architecture: NGO + EOS vs competitors' broken netcode

2. **🔧 Esports-Grade Networking**
   - Region-optimized matchmaking eliminates lag issues plaguing Overcooked 2
   - Ping filtering and smart host selection
   - Built for competitive play, not just casual co-op

3. **🤖 Intelligent Bot Filling**
   - No waiting for full lobbies - AI fills empty slots immediately
   - Matches always available, 24/7
   - Scales from 2-8 players seamlessly

4. **⚖️ Fair & Balanced Gameplay**
   - Character abilities create strategic depth without unfair advantages
   - Progression systems reward skill, not spending
   - Competitive integrity prioritized

5. **📱 True Cross-Platform Potential**
   - Architecture supports mobile ↔ PC ↔ console play
   - Friends can play together regardless of device
   - Unified player base for faster matchmaking

6. **🔄 Always Available Multiplayer**

- **Effect:** Pulls all loose ingredients within 5 units to player's position over 3 seconds
- **Cooldown:** 25 seconds
- **Duration:** 3 seconds (ingredients move gradually)
- **Strategy:** Gather scattered ingredients quickly, save time on runs
- **Visual:** Magnetic field aura, ingredients float toward player
- **Audio:** Magnetic hum + item collection sounds
- **Network:** Server controls ingredient positions, clients interpolate
- **Edge Cases:** If player picks up item mid-magnet, stop moving that item
- **Implementation:** Needs area detection, lerping movement, collision avoidance

##### 4. InstantCook (Time Saver)
- **Effect:** Next item placed in cooking station completes instantly
- **Cooldown:** 30 seconds
- **Duration:** Single use (expires after 10 seconds if not used)
- **Strategy:** Emergency order completion, combo optimization
- **Visual:** Cooking station glows gold, sparkles on instant completion
- **Audio:** Magic chime on activation, satisfying "ding" on instant cook
- **Network:** Server validates cooking state, applies instant completion
- **Balance:** Only works on player's interactions (can't steal others' cooking)
- **Implementation:** Flag in CookingStation, check on PlaceItem

##### 5. FreezeTime (Strategic)
- **Effect:** All order timers pause for 4 seconds
- **Cooldown:** 45 seconds (longest cooldown)
- **Duration:** 4 seconds
- **Strategy:** Save failing orders, buy time for complex dishes
- **Visual:** Blue time-freeze overlay on UI, clock icon on orders
- **Audio:** Time-warp sound effect
- **Network:** Server pauses OrderManager timers, syncs to all clients
- **Balance:** Doesn't pause match timer, only order expiration
- **Implementation:** Modify OrderManager.Update() to check freeze flag

### Bot AI Behavior Design

**Current State:** Bots move randomly, no station interaction (20% complete)
**MVP Target:** Bots complete full cooking loop (70%+ match completion)

#### Bot Architecture

```
BotController (existing)
├── BotDecisionEngine (NEW)
│   ├── Perception System
│   ├── Goal Selector
│   └── Action Executor
└── BotMemory
    ├── Known orders
    ├── Station states
    └── Ingredient positions
```

#### Bot Decision Making (State Machine)

**States:**
1. **IDLE** - Waiting for orders
2. **GATHERING** - Getting ingredients from crates
3. **PREPARING** - Cutting/cooking at stations
4. **ASSEMBLING** - Putting ingredients on plates
5. **SERVING** - Delivering to serving station
6. **SABOTAGING** - (Future: PvP actions)

#### Bot Behavior Logic

**Priority System (Weighted Random):**
```
1. URGENT (60% weight)
   - Order expires in <15 seconds
   - Have all ingredients for this order
   → Rush to serve

2. HIGH (30% weight)
   - Have partial ingredients for active order
   → Continue preparing this order

3. MEDIUM (8% weight)
   - No current order
   → Start easiest available order

4. LOW (2% weight)
```

### Technical Implementation Notes

#### Ability System Architecture
```csharp
// Network-synced ability activation
[ServerRpc]
void ActivateAbilityServerRpc(AbilityType type)
{
    if (!CanActivateAbility(type)) return;
    
    var ability = GetAbility(type);
    if (ability.Activate())
    {
        NotifyClientsAbilityActivatedClientRpc(type);
        StartCooldown(type);
    }
}

// Validation prevents cheating
bool CanActivateAbility(AbilityType type)
{
    return !IsOnCooldown(type) && 
           !IsStunned && 
           HasUnlockedAbility(type);
}
```

#### Bot AI Performance
- Update tick: 10Hz (0.1s) to save CPU
- Pathfinding: Use Unity NavMesh or A* grid
- LOD System: Simplify AI for distant bots
- Culling: Pause AI for bots not near players

#### Network Optimization
- Ability activation: ServerRpc (0.1KB)
- Bot decisions: Server-only (no network traffic)
- State sync: NetworkVariables (automatic)
- Position: Client prediction + server reconciliation

#### Memory Management
- Object pooling for ingredients/particles
- Bot limit: Max 4 bots per match (memory/CPU)
- Station state: Only sync when changed
- Ability effects: Visual-only on clients (server validates)
